GC Music Teaching Hub Submission Processing

Submission Workflow

  1. Someone submits materials to the website, and the bot posts in the Slack channel #submissions.
  2. React with an emoji to the bot’s message, indicating that you’ve taken responsibility for adding that submission to the website.
  3. Review the submission
    • On the review page, click on the buttons to navigate between submissions. It will default to the latest one.
    • Review the submission before proceeding. You can check the documents by clicking on the links
  4. Prepare the submission for upload. There are some measures that need to be taken to ensure the submission is fully openly licensed and to remove barriers that might prevent people with disabilities from accessing the submission.
    • On the document processing page, add links to Google Drive documents if necessary and click the Submit and Download button. This will convert DOCX files to PDFs, OCR any PDFs, and download all of the documents. This may take a few seconds.
    • Open LibreOffice Draw to edit the pdfs. It can be found as a free download here.
    • Add the Creative Commons License as an image to every page of every file. You can find them here here. We default to BY NC SA.
    • Add alternative text to any images, including the Creative Commons License, by right-clicking on an image and clicking description. Write the text in the “Text Alternative” box.
    • If there are images on the document, make sure they were created by the doc's author, or that the image is openly licensed (e.g., from Wikimedia). If not, then the image needs to be removed (i.e., place a black box over it), or replaced with an author-created image or other image that is openly licensed.
    • If the document has any text or images that are low color-contrast, this needs to be addressed - either by adjusting the color contrast of the item or by making a note in the post that the artifact requires vision for this reason.
    • If the document has any text in which font styling or color conveys crucial information, this needs to be addressed, either by changing the font styling or color and conveying this information another way, or by making a note in the post that the artifact requires vision for this reason.
    • If the document has any links in it, check to see if the link text is descriptive. If not, change the link text so that it is descriptive, or make a note in the post that the artifact requires vision for this reason.
  5. Upload the file and post
    • On the upload page, upload the finished PDFs
    • You'll be redirected to a new page.
    • Fill in any empty fields and check the pre-filled fields before clicking on the submit button at the bottom of the page.
    • You'll be redirected to the email form.
  6. Send the submitter an email
    • Fill out the form here.
  7. Add the post to TablePress
    • Log into the commons.
    • Click here to reach our table
    • Scroll down until you see the end of the table.
    • Add a row using the button on the left within "Table Manipulation"
    • Populate the new row
    • To add a new link, click "Insert Link" and follow the instructions. Link Text is what you want to appear in the table.
    • Test the new entry in the table on the homepage.
  8. Add the submitter’s name to ‘Contributors’ page. List full name (in alphabetical order by last name) and subdiscipline.
